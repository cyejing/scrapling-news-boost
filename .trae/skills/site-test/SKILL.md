# 网站抓取测试技能

用于验证 `scrapling-web-fetch` 技能对各主流新闻网站的抓取能力。

## 测试目标

- 文章内容包含标题、正文
- 内容基本完整
- 结构化输出，便于大模型理解
- 清除噪音，减少 token 消耗
- 绕过反爬机制

## 测试流程

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 执行测试 site_test.py fetch                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. 保存测试结果 site_test.py save-result                         │
│                                                                 │   
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. 更新测试用例 site_test.py update-case                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. 生成汇总报告                                                  │
│     汇总本次测试结果 → 输出到 output/test_report.md                 │
└─────────────────────────────────────────────────────────────────┘
```

## 命令用法

注意: 当用户没有明确要求重复测试已通过的用例，默认仅测试状态为 `failed` 或 `timeout` 的用例。

### 1. 执行测试 (fetch)

**参数说明**：

| 参数      | 必填 | 说明                                         |
|-----------|------|----------------------------------------------|
| `--id`    | 否   | 指定测试用例编号，多个用逗号分隔              |
| `--all`   | 否   | 测试所有用例，包括已通过的                    |
| `--output`| 否   | 输出到 JSONL 文件，不指定路径则使用默认路径    |

**用法示例**：

```bash
# 测试所有未通过的用例（状态非 passed）
uv run scripts/site_test.py fetch

# 测试指定编号的用例
uv run scripts/site_test.py fetch --id 02,05,08

# 全局测试（包括已通过的用例）
uv run scripts/site_test.py fetch --all

# 输出到默认文件 output/test_fetch_all_results.jsonl
uv run scripts/site_test.py fetch --all --output

# 输出到指定文件
uv run scripts/site_test.py fetch --all --output output/custom_results.jsonl
```

**输出格式**：

```json
{
  "id": "02",
  "domain": "finance.sina.com.cn",
  "url": "https://...",
  "fetch_result": {
    "ok": true,
    "title": "文章标题",
    "content_length": 5000,
    "fetch_mode": "stealth",
    "parser": "trafilatura",
    "total_duration": 5.56,
    "content": "正文内容..."
  }
}
```

### 2. 保存测试结果 (save-result)

**参数说明**：

| 参数              | 必填 | 说明                               |
|-------------------|------|------------------------------------|
| `--id`            | 是   | 用例编号                           |
| `--status`        | 是   | 状态：passed/failed/timeout/pending |
| `--score`         | 是   | 质量评分（0-100）                  |
| `--comment`       | 是   | 质量评价                           |
| `--fetch-mode`    | 是   | 抓取模式：stealth/fetcher/urllib   |
| `--parser`        | 是   | 解析器：trafilatura/scrapling      |
| `--content-length`| 是   | 文本长度                           |
| `--total-duration`| 是   | 总耗时（秒）                       |
| `--title`         | 是   | 文章标题                           |
| `--suggestion`    | 是   | 优化建议                           |

**用法示例**：

```bash
uv run scripts/site_test.py save-result \
  --id 02 \
  --status failed \
  --score 10 \
  --comment "内容全为噪音" \
  --fetch-mode stealth \
  --parser trafilatura \
  --content-length 150 \
  --total-duration 2.95 \
  --title "文章标题" \
  --suggestion "尝试抓取对应 html 节点..."
```

### 3. 更新测试用例 (update-case)

**参数说明**：

| 参数      | 必填 | 说明                               |
|-----------|------|------------------------------------|
| `--id`    | 是   | 用例编号                           |
| `--status`| 是   | 状态：passed/failed/timeout/pending |
| `--score` | 是   | 质量评分（0-100）                  |
| `--comment`| 是  | 质量评价                           |

**用法示例**：

```bash
uv run scripts/site_test.py update-case \
  --id 02 \
  --status failed \
  --score 10 \
  --comment "内容全为噪音"
```


## 大模型评估指南

### 评估步骤

1. **查看抓取结果**：检查 `content` 字段的内容
2. **判断状态**：
   - 抓取成功且内容完整 → `passed`
   - 抓取成功但内容有问题 → `failed`
   - 抓取超时（>30秒）→ `timeout`
3. **质量评分**：根据质量评分标准打分
4. **质量评价**：简述问题或优点

### 质量评分标准（0-100分）

| 分数区间 | 说明 |
|----------|------|
| 90-100 | 内容完整，噪音清除干净 |
| 70-89 | 内容基本完整，有少量噪音 |
| 50-69 | 内容有缺失或噪音较多 |
| 0-49 | 抓取失败或内容严重缺失 |

### 状态枚举值

| 枚举值 | 显示文本 | 说明 |
|--------|----------|------|
| `passed` | ✅ 通过 | 跳过测试 |
| `failed` | ❌ 失败 | 需要测试 |
| `timeout` | ⚠️ 超时 | 需要测试 |


## 测试汇总报告

测试完成后，汇总所有测试结果到 `output/test_report.md`：

### 汇报模板

```

**测试时间**：2026-03-16  
**测试用例数**：10 条

## 测试汇总
| 编号 | 原始 URL | 抓取状态 | 文本长度 | 质量评分| 质量评价 | 变化 | 优化质量 |
|------|----------|----------|----------|----------|------|----------|
| 01 | mp.weixin.qq.com | ✅ 通过 | 5234 | 95 | 内容完整，噪音清除干净 | 否 | 0 |
| 02 | finance.sina.com.cn | ❌ 失败 | - | 10 | 内容全为噪音 | 否 | 0 |

## 统计数据

- **通过率**：7/10 (70%)
- **平均质量评分**：61.0 分
- **平均抓取耗时**：2.70 秒

## 问题分析

### 失败用例

| 编号 | 域名 | 问题 | 优化建议 |
|------|------|------|----------|
| 02 | finance.sina.com.cn | 抓取到的是Cookie政策文本，正文内容完全缺失 | 尝试使用 scrapling 解析器或调整选择器 |

### 通过用例
| 编号 | 域名 | 质量评价 |
|------|------|----------|
| 01 | mp.weixin.qq.com | 内容完整，噪音清除干净 |
| 06 | www.sohu.com | 内容基本完整，噪音清除较好 |

```

**字段说明**：
- **变化**：相较于上一次测试结果是否有变化
- **优化质量**：相较于上一次测试的优化程度（-100 到 +100）
